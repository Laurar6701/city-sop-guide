Revision rule: While writing or editing, always name the file RevROUGH. Once approved, rename it to the next revision number (e.g. Rev3). Never skip revision numbers. Never use decimals like Rev3.5.

Function naming rules
  • Use 1–2 words maximum — clear and easy to understand across departments.
  • Avoid abbreviations unless widely recognized company-wide (e.g. CFOS, SHRM).
  • The function groups related processes — it is not the specific task itself.
  • Department leaders may create new function names as needed. Keep them clear and reusable.
Step 2 of 4

Building an SOP

Follow these steps in order. All draft work happens in the Drafts (In-Progress) folder — never in the published library.

Draft SOPs folderSharePoint — where you work
1

Get the SOP template

Navigate to the Draft SOPs folder above. Go to Document Management → SOPs (Rough Draft) → Draft SOPs → Templates and copy the template file.

  • Windows Explorer: Right-click → Copy → navigate to your department's Drafts (In-Progress) folder → Right-click → Paste.
  • SharePoint web: Right-click → Copy to → navigate to your department's Drafts (In-Progress) folder → Copy here.

If you are revising an existing SOP, copy it from Drafts (Release) into Drafts (In-Progress) instead.

2

Rename the file

Before you start writing, rename the file following the SOP Naming Convention (see the Naming tab). Use RevROUGH as the revision until it is approved.

Example: AD – Document Control – Onboarding Process – RevROUGH

3

Fill in the header

Complete every field in the SOP header box:

  • Department — use your approved department code.
  • Function — 1–2 word category.
  • Date Issued — leave blank while the SOP is in draft.
  • Procedure (Name) — under 8 words, precise enough for a reader to know what it covers at a glance.
  • Procedure # — follows the naming convention.
4

Write the SOP content

Follow these writing standards:

  • Start each step with an action verb (e.g. Open, Select, Navigate, Complete).
  • Use present tense throughout — never past tense.
  • Spell out numbers (one, two, three — not 1, 2, 3).
  • No contractions — write "do not" not "don't."
  • Each section heading must use Heading 3 style so it appears in the Table of Contents.
  • Direct all steps to one person — use role titles, never names.
5

Update the Table of Contents

Right-click anywhere on the Table of Contents → Update Table → Update entire table → OK.

If unexpected items appear, find those paragraphs and change their style from "Heading 3" to Normal.

6

Update the Revision History table

At the bottom of the SOP, fill in the Revision History table with the date, revision number, a brief description of changes, and your initials. Revision numbers are always whole numbers starting at 1 — never skip numbers.

7

Final rename before submitting

Once the SOP is approved by your leader, rename the file: change RevROUGH to the correct revision number (e.g. Rev3). Update the Date Issued in the header to today's date. Confirm the file name matches the Procedure # in the header exactly.

Step 3 of 4

Submitting for publication

Once your leader has approved the SOP, follow these steps to get it published. You do not publish it yourself — OSM publishes on your behalf.

Review before you submit. Make sure your SOP has been reviewed with your leader and all applicable stakeholders before submitting. Once submitted, allow 2–3 business days for publication.

1

Review with your leader and stakeholders

Do not submit until the SOP has been reviewed and approved by your direct leader and any other applicable stakeholders who own or perform the process.

2

Open the Publishing Request Form

Click the button below to open the form. Complete all required fields.

Open Publishing Request Form
3

Submit and wait

After submitting, allow 2–3 business days for the SOP to be published to the official library.

4

Submit UKG requests (if applicable)

If the SOP requires any UKG updates (training, role assignments, etc.), submit those requests after the publication is approved. Allow a minimum of two weeks for UKG requests to be completed.

Step 4 of 4

Finding published SOPs

All approved, current SOPs live in one place. Always use this library — do not rely on saved copies, printed versions, or drafts you may have downloaded.

Published SOP Library

This is the only source of truth for current, approved SOPs. Bookmark this link and use it every time you need to reference a procedure.

Open Published SOP Library Draft SOPs folder
How to search
Use the search bar in SharePoint to find SOPs by department code, function name, or key words in the procedure name. Results only show published documents in this library.
Reading the revision number
The file name always ends in a revision number (e.g. Rev5). The highest number is always the most current. If you have a printed or downloaded copy, verify its revision matches the library before using it.
Important

Avoid using archived SOPs

Archived SOPs are outdated versions that have been replaced. Using them can lead to errors, compliance issues, and inconsistent practices across teams.

Never use a document labeled "Archive" or "RevROUGH" to perform or reference a process. These are not current versions and may contain outdated or incorrect information.

How to verify you have the current version
  • Always access SOPs directly from the Published SOP Library (see the Finding SOPs tab).
  • Check the file name — the revision number at the end (e.g. Rev5) should match the highest revision in the library for that SOP.
  • If you have a printed copy, compare its revision number to the current version in the library before using it.
  • If you are unsure whether a version is current, do not use it — open the library and download a fresh copy.
What to do if you find an archived SOP being used
  • Do not distribute or act on it.
  • Locate the current version in the Published SOP Library and share that link instead.
  • Notify the SOP owner or your department leader so the archived copy can be removed or corrected.
